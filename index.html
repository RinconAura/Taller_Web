TypeScript

¿Qué es?

TypeScript es un superconjunto de JavaScript que añade tipado estático opcional y funciones avanzadas a JavaScript, que permite a los desarrolladores especificar los tipos de variables, parámetros de función y valores de retorno, detectando errores relacionados con el tipo durante el desarrollo.

Sintaxis

  • Declaración de variables:

    • Una variable es un contenedor que almacena un valor, y ese valor puede cambiar durante la ejecución del programa. Las variables se utilizan para guardar datos temporales que pueden ser referenciados y manipulados.

    Hay tres formas principales de declarar variables:
    1. var: Tiene un alcance a nivel de función y puede ser redeclarada.
    2. let: Permite declarar variables con un ámbito limitado a un bloque de código (bloque de función, bloque if, etc.).
    3. const: Declara variables cuyo valor no puede cambiar una vez asignado, aunque en el caso de objetos y arrays, se pueden modificar sus propiedades o elementos internos.

let nombre: string = "Juan";
const PI: number = 3.1416;


  • Definición de funciones:

    • Una función es un bloque de código que se agrupa para realizar una tarea específica y se puede reutilizar cuando se necesite. Las funciones permiten evitar la repetición de código, hacen que el código sea más legible y estructurado, y permiten la modularidad.

    function suma(a: number, b: number): number { return a + b; }

  • Interfaces:

    • En TypeScript, una interfaz define la estructura de un objeto, especificando los tipos de las propiedades y métodos que debe tener.

    interface Persona {
    nombre: string;
    edad: number;
    saludar(): void;
    }

  • Types:

    • Los types en TypeScript son similares a las interfaces, pero más flexibles. Mientras que una interfaz solo puede definir la estructura de objetos, un type puede representar no solo objetos, sino también tipos primitivos, uniones, intersecciones, funciones, tuplas, etc..
    Los types pueden unirse usando & (intersección) y | (unión), lo que no es posible con interfaces.


    type Punto = { x: number; y: number; }
    type ID = number | string;

    Variables y tipos

    • Tipos
      • Primitivos principales
      • 1. boolean : valores verdaderos o falsos
        let esEstudiante: boolean = true;
      • 2. number : números enteros y valores de punto flotante
        let edad: number = 10;
      • 3. string : valores de texto como "TypeScript Rocks"
        let nombre: string = "Pepito";

      • Primitivos menos comunes
      • 1. bigint : números enteros y valores de punto flotante, pero permite números negativos y positivos más grandes que el numbertipo.
      • 2. symbolSe :utilizan para crear un identificador único global.

      • Tipos especiales
      • 1. any : es un tipo que deshabilita la verificación de tipos y permite efectivamente que se utilicen todos los tipos.
        let v: any = true; v = "string"; Math.round(v);
      • 2. unknown :unknownes una alternativa similar, pero más segura, a any.

      • 3. never :efectivamente lanza un error cada vez que se define.
        let x: never = true; // Error: Type 'boolean' is not assignable to type 'never'.
      • 4. undefined & null :son tipos que hacen referencia a los primitivos de JavaScript undefinedy null respectivamente.
        let y: undefined = undefined; let z: null = null;


    Clases

    • Propiedades y Métodos:
      se tipifican mediante anotaciones de tipo, similares a las variables.
      class Person {
      name: string;
      }
      const person = new Person();
      person.name = "Jane";

    • Visibilidad :
      Hay tres modificadores de visibilidad principales en TypeScript.
    • 1. public : (predeterminado) permite el acceso al miembro de la clase desde cualquier lugar
    • 2. private : solo permite el acceso al miembro de la clase desde dentro de la clase
    • 3. protected : permite el acceso al miembro de la clase desde sí mismo y desde cualquier clase que lo herede, lo cual se cubre en la sección de herencia a continuación

      • class Person {
      private name: string;
      public constructor(name: string) {
      this.name = name;
      }br public getName(): string {
      return this.name;
      }
      }
      const person = new Person("Jane");
      console.log(person.getName()); // person.name isn't accessible from outside the class since it's private

    Ciclos y condicionales

    • 1. Condicionales:

      • Los condicionales permiten que un programa tome decisiones ejecutando diferentes bloques de código en función de si una condición es verdadera o falsa. En la mayoría de los lenguajes de programación, los condicionales se implementan utilizando las declaraciones if, else if, y else.

      if (x > 10) { console.log("Mayor que 10"); }
    • 2. Bucle for:

      • Se utiliza para repetir un bloque de código un número específico de veces.

      for (let i = 0; i < 5; i++) { console.log(i); }
    • Bucle while:

      • Un bucle while repite un bloque de código mientras una condición específica sea verdadera.

      while (x < 10) { x++; }

    Arrays y Listas

    • Declaración de Arrays:

      • Un array es una estructura de datos que permite almacenar múltiples valores en una sola variable, organizados en una secuencia indexada. La declaración de un array define la capacidad de almacenar y acceder a varios elementos utilizando índices.

      let numeros: number[] = [1, 2, 3, 4, 5];
    • Uso de métodos:

      Los métodos son funciones que están asociadas a un objeto o clase. Se utilizan para realizar operaciones o manipular los datos de ese objeto. Los métodos se llaman utilizando el nombre del objeto seguido del nombre del método y, en caso necesario, se pueden pasar argumentos entre paréntesis.

      numeros.push(6);
      numeros.map(num => num * 2);

    Angular

    ¿Qué es?

    Angular es un framework de desarrollo web creado por Google para construir aplicaciones web dinámicas de una sola página (Single Page Applications, SPA). Angular utiliza TypeScript como su lenguaje principal y ofrece herramientas y estructuras para crear aplicaciones robustas y mantenibles.


    Configuración

    • Instalación de Angular CLI: npm install -g @angular/cli
    • Creación de un nuevo proyecto: ng new mi-proyecto
    • Iniciar el servidor de desarrollo: ng serve

    Características principales:

    • 1. Componentes: Angular se basa en una arquitectura de componentes, que facilita la reutilización y el mantenimiento del código.
    • 2. Inyección de dependencias: Facilita la gestión de dependencias entre los distintos componentes y servicios de la aplicación.
    • 3. Routing: Angular tiene un sistema de enrutamiento robusto para manejar la navegación entre diferentes vistas en una SPA.
    • 4. Directivas y Pipes: Permiten manipular el DOM y transformar datos en las plantillas de forma declarativa.

    Ionic

    ¿Qué es?

    Ionic es un framework open-source utilizado para desarrollar aplicaciones móviles híbridas, es decir, aplicaciones que pueden ejecutarse en iOS, Android y web desde un solo código base. Ionic está construido sobre Angular (aunque también soporta React y Vue) y utiliza tecnologías web estándar como HTML, CSS y JavaScript.


    Configuración

    • Instalación de Ionic CLI: npm install -g @ionic/cli
    • Creación de un nuevo proyecto: ionic start mi-proyecto blank
    • Iniciar la aplicación: ionic serve

    Características principales:

    • UI Components: Proporciona una amplia gama de componentes de interfaz de usuario optimizados para dispositivos móviles.
    • Cross-platform: Permite crear aplicaciones que funcionan en múltiples plataformas sin necesidad de escribir código específico para cada una.
    • Integración con Cordova/Capacitor: Permite acceder a las funcionalidades nativas del dispositivo como la cámara, GPS, etc.
    • Theming: Soporte para personalización de temas, adaptándose a la apariencia de cada plataforma (iOS, Android).
